# 経歴

## アプリデータの転記業務

- 使用技術…Excel、VBA
- 業務詳細
  - アプリ内のデータをITの知識が無い人でも分かるように転記する業務
  - DBの関連などを図にして表したり、必要な情報のみを抜き取って整理して記述する
- 自主学習
  - Excelの関数やVBAの使い方などを学習し、作業を自動化する方法を開発し、メンバーに共有した（VBAはセキュリティの関係で実際の業務では使えなかった）
    - 背景…繰り返し行う定型作業があり、これを自動化し、作業のスピードを上げたいと考えた

## 自社の営業ツールサイトの開発

- 使用技術
  - 言語…PHP
  - フレームワーク…Laravel, React
  - DB…MariaDB
  - OS…Linux(Ubuntu)
  - 補足…AWS上でアプリを運用していた
- 業務詳細
  - 自社の営業部が使用する社員のスキルシート情報を管理するサイトの開発（機能追加、修正、変更）
  - 営業側から大雑把な機能追加・変更の要望があり、それを元に必要な設計や画面遷移図を作成し、上司と相談の後承認されたら、開発→テスト→デプロイを行った
  - 後半は今後追加されるメンバーに向けたドキュメント作成、後輩のコードのレビュー等を行った

- 自主学習
  - 1 チームメンバーの業務を支援するツールの開発（チャットアプリの特定のメンバーの特定の期間のチャット内容を一括取得するアプリ）
    - 背景
      - 上司が朝の~時までにメンバーが朝の報告を行っているかどうかで遅刻の判定を行っていた
      - 確認すべきメンバーの数が50人程と多く、毎日その作業を手作業で行っており、これを支援できないかと考えた
    - Chrome拡張機能版…DOM操作でチャットを一括取得
    - デスクトップアプリ版（Python）…チャットアプリのAPIを使用しチャットを一括取得
    - Google App Script & スプレッドシート版…指定日時になったら自動でチャット内容を一括取得し、スプレッドシートに日毎のチャット内容を挿入し、遅刻した場合は色で分かりやすくする機能を開発
  - 2 チーム開発で行っているデプロイ時の定型作業を自動化するシェルスクリプトの実装
    - 背景
      - デプロイ時は変更したコードを直接上書きする方針で行っていた
      - 変更したファイルの追跡やファイルのバックアップなどを全て手作業で行っており、自動化できないかと考えた
    - シェルスクリプトの使い方から学習し実装した（最初はJavaScriptで実装していたが、Node環境が無い人でも実行させる必要がありシェルスクリプトで実装し直した）

### 離職理由

- 前職での自社WEBサイト開発でメンバーが少なく中核的な人材になっていた。そのためチーム側があまり派遣先に出したくない意向があった。
- 途中からWEBサイトの機能要望が少なくなり、会社の雑務（PCの設定等）や、新しくチームに入る人に向けたドキュメント作成が中心になり、相談を行ってはいたがなかなか開発に戻れたり、派遣先に行けるような雰囲気にならなかった。そのため会社を離職することを決断した。
